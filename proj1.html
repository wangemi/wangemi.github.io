Campus Maps

LANGUAGES: Java, Javascript, React, HTML/CSS
CLASS PROJECT: Software Design & Implementation
DURATION: 6 weeks
SPRING 2019

Campus Maps is web application that allows students to quickly find the shortest path between two buildings on campus. Simply input two buildings of your choice, and Campus Maps will show you a visual path that you can take!


The Focus

The bulk of this project centered around building an effective graph ADT that can support a decent amount of data without efficiency loss.

This project also focused on developing pragmatic programming skills, such as learning how to properly document code, write programs that are correct, easy to modify, understand, and scale. In addition, I learned how to write testing code for my graph ADT, and use Javascript, React, and HTML/CSS for the final front facing web application.

The Process

— PART 1
First, I specified the API for a directed labeled graph intended to support the campus map. I thought about operations my ADT would need, and broke that down into classes / interfaces I would need. After writing out the high level specification, I proceeded to write the skeleton code of what I would need, documenting each class and method stud with Javadoc.

With my skeleton code completed, I then wrote both specification and implementation tests. Tests were written before actually implementing the ADT because this allows me to know exactly how my code should function prior to implementing it.

— PART 2
During the second part of the project, I implemented my graph ADT. I chose to use a collection of Edges to represent my graph. I focused primarily on maintaining good style and design during the early stages, though I made sure my code was still reasonably efficient. I further documented my code with abstraction functions and representation invariants to practice pragmatic programming.

— PART 3
With my completed graph ADT, it was time to test the scalability of my implementation. The data provided to me was the how the Marvel universe's characters related to each other, and contained thousands of nodes and edges. I fixed any issues that impacted performance and efficiency at this stage.

— PART 4
Now that my graph ADT runs smoothly and efficiently, I made my graph ADT generic, and added the path finding algorithm (Dijkstra's) that would be needed for the final web application.

— PART 5
The final part of this project was to use Javascript, React, and some HTML/CSS to create the front end of the Campus Maps web application.
