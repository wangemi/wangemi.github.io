Zip Utility

LANGUAGE: Java
CLASS PROJECT: Data Structures & Parallelism
PARTNER PROJECT
DURATION: 2 weeks
AUTUMN 2019

The zip utility zips given input files. Some accepted formats include PDFs, HTML files, and .txt files.


The Focus

This project focused on implementing work lists for the backend of the zip utility. Some work lists that we coded include LIFO queues, array stack, min-heap, and hash trie sets.

This is the first project out of the three for the Data Structures and Parallelism class. Code written in this project is built upon in a later project, uMessage.

The Process

— PART 1
For the first part of this project, we implemented work lists such as the queues and stacks. For the queues, we had two different implementations — one used a linked list, and the other used a circular array.

— PART 2
Next, we wrote a min heap as array-based that starts at index 0. Our min heap is a four heap with four children per node. We also coded a HashTrieMap and HashTrieSet as well.

— PART 3
Now that our standard work lists are completed, we ran the provided Zip client code. The Zip client used the min heap / PriorityQueue to do the Huffman coding, the FIFO queue as the buffer, the stack to calculate the key set of a trie, and the SuffixTrie for LZ77Compression.
